Pages from files

Route real pages straight from your folder tree.

Roselt.js resolves pages, scripts, styles, and metadata from matching files so the structure of the app stays readable as the project grows.

You can start with one entry page, add nested guides or app views, and keep the route map obvious without maintaining a separate route registry.

Persistent shell

Keep the frame mounted while routed content swaps cleanly.

Shared sections live in the shell, so the header, sidebar, and footer remain stable while navigation updates only the routed page content.

That makes Roselt.js a good fit for full product applications, internal tools, documentation portals, and teams that want direct HTML authoring without giving up app structure.

Flexible delivery

Deploy where static delivery fits, and package where installs matter.

The default query-based router works cleanly on static hosts, and pathname routing is available when your host can rewrite every request back to the entry page.

Roselt.js stays close to the platform: HTML files, module scripts, custom elements, and metadata controls that are easy to inspect, reason about, and reuse across serious application surfaces.

0 runtime dependencies in the published package
1 root entry page required to start a routed app
3 core directories to remember: pages, sections, and components
SPA navigation that still fits naturally on static infrastructure
