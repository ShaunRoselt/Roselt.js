Metadata and SEO

Set page titles, descriptions, canonicals, and social text with the page module.

Roselt.js updates the document title, canonical URL, description, Open Graph text, and Twitter text after each page render. The framework keeps the metadata surface small and route-driven.

Metadata you can set today

Roselt.Page.Title = "Home";
Roselt.Page.Description = "Home page";
Roselt.Page.Canonical = "?page=home";

When only title and description are present, Roselt.js automatically uses them as the fallback values for og:title, og:description, twitter:title, and twitter:description.

How metadata is resolved

  1. Route-level metadata is applied first when you define explicit routes.
  2. The page module metadata from Roselt.Page.* overrides route metadata.
  3. Metadata returned from Roselt.Page.Load(...) as part of mountResult.meta wins last.
Roselt.Page.Load = () => {
  return {
    meta: {
      title: "Dashboard",
      description: "Live dashboard data for the current account."
    }
  };
};

Canonical handling

If you do not set Roselt.Page.Canonical, Roselt.js uses the current URL as the canonical. That means query-based docs pages keep their route query parameter in the canonical link by default.

Static-host fit

For a query-routed GitHub Pages site, that default is usually exactly what you want: each doc page remains directly shareable and canonicalized to the URL users actually load.

What Roselt.js intentionally does not do

  • No server-side rendering layer.
  • No framework-specific sitemap or robots system.
  • No markdown or content pipeline hidden behind the metadata API.
