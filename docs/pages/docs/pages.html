Pages

Each page can ship markup, metadata, behavior, and same-name CSS.

Roselt.js pages are intentionally plain: an HTML file for markup, an optional JavaScript file for metadata and lifecycle behavior, and an optional CSS file loaded automatically from the same directory.

HTML first

The HTML file is the route's primary source of content. Inline style tags are extracted and applied before the page swaps in, which helps avoid flicker during navigation.

<section data-page-home>
  <h1>Home</h1>
  <p>This route came from pages/home.html.</p>
</section>

Page module contract

Page modules run when their matching route module is loaded. Set metadata at the top level, then use Roselt.Page.Load only when the rendered HTML needs behavior.

Roselt.Page.Title = "Home";
Roselt.Page.Description = "Home page";

Roselt.Page.Load = () => {
  const button = Roselt.Page.querySelector("[data-next-page]");

  if (!(button instanceof HTMLButtonElement)) {
    return null;
  }

  button.setAttribute("href", Roselt.Page.href("about"));

  const handleClick = () => Roselt.Page.navigate("about");

  button.addEventListener("click", handleClick);

  Roselt.Page.cleanup(() => {
    button.dataset.ready = "false";
  });
};

Roselt.Page.Load does not need a context argument. The active page state and helpers are available directly from Roselt.Page while the route is mounted.

What Roselt.Page exposes at runtime

Property Meaning
Roselt.Page.querySelector() and Roselt.Page.querySelectorAll() DOM query helpers scoped to the routed page root after the new page HTML has been inserted.
Roselt.Page.route and Roselt.Page.params The matched route object and any pathname parameters for explicit routes.
Roselt.Page.url and Roselt.Page.query The current URL object and a plain-object view of the search params.
Roselt.Page.href() and Roselt.Page.navigate() Helpers for building internal URLs and performing programmatic navigation.
Roselt.Page.cleanup() Register additional cleanup callbacks to run when the page unmounts.
return cleanup or return { cleanup, destroy, meta } Return page teardown work, runtime metadata, or both from Roselt.Page.Load.
Roselt.Page.notFound Provides the failed URL details when Roselt.js is rendering a custom 404 page.

Automatic cleanup

Event listeners added during Roselt.Page.Load(...) are automatically tracked through an abort signal wrapper. You can still return a cleanup function or register explicit cleanup callbacks for libraries, observers, timers, or anything outside normal event binding.

Practical rule

If your page module only sets metadata, the file can be just two lines. Add a Load handler only when you need DOM behavior.
