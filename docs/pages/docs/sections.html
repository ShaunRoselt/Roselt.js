Sections

Sections are reusable HTML fragments that can stay mounted across route changes.

Use sections for shared shell fragments such as headers, footers, sidebars, legal notices, or cross-page utility UI. Roselt.js resolves them recursively and caches them after startup.

Load a section with shorthand

<roselt section="site-header"></roselt>

Roselt.js resolves shorthand names like site-header to sections/site-header.html by default.

Optional section sidecars

Sections can now ship same-name sidecar files, similar to pages. When sections/site-header.html is used, Roselt.js also tries sections/site-header.css and sections/site-header.js automatically.

sections/
  site-header.html
  site-header.css
  site-header.js

Use the CSS file for shared section styling and the JS file for section-level browser behavior such as search overlays, menus, or shell controls.

Resolve sections recursively

Section HTML can include other sections. Roselt.js processes the fragment as a template and resolves nested roselt section elements before inserting the final markup.

<!-- sections/site-header.html -->
<header>
  <roselt section="brand-lockup"></roselt>
  <nav>...</nav>
</header>

Where sections work best

Persistent shell UI

Wrap your page root with shared header and footer sections in index.html so they resolve once and remain mounted during navigation.

Portable content blocks

Use sections as reusable content fragments for banners, shared calls to action, or repeated editorial modules.

App-level composition

Because sections resolve before routed content renders, they are a good place for shared layout scaffolding that should not reinitialize on each route.

Path resolution rules

Reference How Roselt.js resolves it
site-header Resolved relative to the configured sectionsDirectory as sections/site-header.html, with optional sections/site-header.css and sections/site-header.js sidecars loaded automatically.
shared/promo.html Treated as an explicit relative path and resolved against the including file.
https://... Treated as a full URL reference.
