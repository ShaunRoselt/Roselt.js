FAQ

Common decisions and tradeoffs.

Roselt.js is small enough that most questions come down to hosting assumptions, directory layout, and how much framework behavior you actually want layered over the browser platform.

Questions developers usually ask first

Does Roselt.js require a build step?

No. The framework can start directly from the prebuilt global bundle. The build scripts in this repository exist to produce the published dist/ outputs.

What does the CDN URL look like now?

Use https://cdn.jsdelivr.net/gh/ShaunRoselt/Roselt.js@main/dist/roselt.js for the browser global and call window.Roselt.start(). The minified browser-global build is https://cdn.jsdelivr.net/gh/ShaunRoselt/Roselt.js@main/dist/roselt.min.js.

Why is query routing the default?

Because it works on static hosts without rewrites. That makes Roselt.js a practical choice for GitHub Pages-backed documentation or product sites.

Can I skip the JavaScript file for a page?

Yes. If a route only needs markup, omit the .js file entirely. Roselt.js treats missing page modules as optional.

Do I have to use custom elements?

No. Components are optional. Plain HTML pages and reusable sections are enough for many Roselt.js sites.

What browser target does Roselt.js assume?

Latest browsers with Navigation API support. The repository builds ES2022 output and treats the browser Navigation API as the routing foundation.
