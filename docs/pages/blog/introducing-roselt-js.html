Introducing Roselt.js

A file-based framework for building full web applications with vanilla JavaScript.

We're excited to announce Roselt.js — a lightweight, file-based framework for building full web applications, product sites, and desktop-ready shells using vanilla JavaScript and real HTML files.

Why Roselt.js?

Modern frameworks solve important problems, but they often pull teams away from the platform. Roselt.js takes the opposite approach: stay close to HTML, keep the runtime small, and let the file system drive the route map.

There's no build step required, no JSX transform, and no virtual DOM. Pages are real HTML files resolved from a pages/ folder. Shared UI like headers and sidebars live in sections/ and stay mounted across navigation. Custom elements live in components/.

Key ideas

  • File-based routing — Drop HTML files in pages/ and the router resolves them automatically.
  • Persistent shell — Sections stay mounted while pages swap, so headers, sidebars, and footers don't flicker.
  • Zero runtime dependencies — The published package ships with nothing extra.
  • Flexible delivery — Works on static hosts out of the box, and can move into packaged shells when platform delivery matters.

Getting started

The fastest way to try Roselt.js is to create an index.html with a page root and a pages/home.html file. Start the router and you have a working routed app. 

<roselt page="home"></roselt>

<script src="https://cdn.jsdelivr.net/gh/ShaunRoselt/Roselt.js@main/dist/roselt.js"></script>
<script>
  window.Roselt.start();
</script>

Head over to the getting started guide for a full walkthrough, or explore the API reference for the complete surface area.
