Roselt.js starter

Ship from files, not framework ceremony.

This starter app is already wired for file-based pages, persistent shell sections, and query-based routing. Start by editing pages/home.html and sections/site-header.html.

Go to Home Page Go to About Page Trigger 404

Where routes live

Every route is just a file inside pages/. Add pages/about.html and navigate to ?page=about.

Keep the shell mounted

Shared chrome belongs in sections/, so headers, footers, and sidebars do not remount on every route change.

Add components from files

Drop a custom element script in components/ and render its tag in your page HTML. Roselt.js will discover and load it automatically.

Add page behavior

Pair markup with pages/home.js when the route needs metadata or client-side behavior.
